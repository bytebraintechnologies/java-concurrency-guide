Thread Synchronization in Java

1. Understanding Thread Synchronization

Pseudocode: Basic Synchronization Concept


// Pseudocode for a shared resource management

// Without Synchronization (UNSAFE)
function updateSharedResource(value):
    // Multiple threads can access simultaneously
    read current value
    modify value
    write modified value

// With Synchronization (SAFE)
function synchronizedUpdateSharedResource(value):
    // Only one thread can access at a time
    LOCK shared resource
    try:
        read current value
        modify value
        write modified value
    finally:
        UNLOCK shared resource

Key Synchronization Principles

  • Prevent multiple threads from corrupting shared data
  • Ensure atomic (indivisible) operations
  • Maintain data consistency in concurrent environments

2. Real-World Scenario: Banking Transaction System

Problem Scenario

  • Multiple bank tellers accessing same account
  • Concurrent withdrawals and deposits
  • Risk of incorrect balance calculations

Synchronization Solution

  • Use synchronized methods
  • Prevent simultaneous account modifications
  • Guarantee accurate balance tracking

public class SecureBankAccount {
    private double balance;
    
    // Synchronized method ensures thread-safe updates
    public synchronized void deposit(double amount) {
        if (amount > 0) {
            // Atomic operation - only one thread can execute
            balance += amount;
            System.out.println("Deposited: $" + amount);
        }
    }
    
    // Another synchronized method for withdrawals
    public synchronized boolean withdraw(double amount) {
        // Check and update in a single, uninterruptible operation
        if (balance >= amount) {
            balance -= amount;
            System.out.println("Withdrawn: $" + amount);
            return true;
        }
        return false;
    }
    
    // Synchronized getter for balance
    public synchronized double getBalance() {
        return balance;
    }
}

// Simulate concurrent banking transactions
public class BankingSimulation {
    public static void main(String[] args) {
        SecureBankAccount account = new SecureBankAccount();
        
        // Multiple threads simulating bank tellers
        Runnable depositTask = () -> {
            for (int i = 0; i < 5; i++) {
                account.deposit(100);
            }
        };
        
        Runnable withdrawTask = () -> {
            for (int i = 0; i < 3; i++) {
                account.withdraw(50);
            }
        };
        
        // Create threads to simulate concurrent access
        new Thread(depositTask).start();
        new Thread(withdrawTask).start();
    }
}

Where Used

Benefits

  • Ensures Accurate Transaction Logging

3. Advanced Synchronization: Concurrent Access Control


import java.util.concurrent.locks.ReentrantLock;

public class AdvancedResourceManager {
    // More flexible locking mechanism
    private final ReentrantLock lock = new ReentrantLock();
    private int resource = 0;
    
    public void safeResourceModification() {
        lock.lock();
        try {
            // Critical section - exclusive access
            resource++;
            System.out.println("Modified resource: " + resource);
        } finally {
            // Always ensure lock is released
            lock.unlock();
        }
    }
}

Advanced Synchronization Techniques

  • Provides more granular control than synchronized methods
  • Allows more complex locking strategies
  • Helps prevent deadlocks and improve performance
